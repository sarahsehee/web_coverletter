Challenge

How to install multiple timers that suit users' lifestyle pattern
during the first setup process.

Survey and interview results showed that 94.9% of users set a timer as soon as they paired smartphones and Swithcer devices for the first time. Also, Swithcer's mobile Google Analytics showed that most of the users don't open the mobile app after the first set-up and just stick with a widget on iOS notification center or a widget on Android Wallpaper.

Willingess to become a paid user by the number of timer set-up

Switcher User Persona

Before we get into the sprint, we refined persona to understand the lifestyle and find the right timer use-case patterns for our users.

User's Pain Points

1. User cannot imagine timer use cases that suit in their lifestyle.
2. There are too many steps to set and save timers.

Design Solution . Version 1

Interactive Onboarding Survey

Switcher User cannot find a sufficient information about timer in the app. After trying a timer Pop-up guide, we realzied that pop-up wasn't enough. So interactive onboarding survey comes in as a solution. It must be feel like consult, smart-home and automattic.

Constraints: The ideal scenario would be incorporate timer function within a widget, but just have a button to open a timer wasn't enough action to encourage users to use the timer. Also, have a timer in widget required more effort and resources than we were limited.

Finding a rigth format to display questionnaire

At this early stage of design process, I used ProtoPie, a codeless prototyping tool to quickly iterate and validate interface and interactions. We've never done any interactive design with many texts in mobile, so it was crucial for us see actionable feedback from user testers.

I used invision to create rough prototypes. These rough prototypes were made for fast iteration and test by myself about timing and the flow of the pop-ups.

Visual Design Exploration

Final Touch

By adding a full screen interactive survey to shift the focus to users' lifestyle, there must be a change to show the timer.
The timer list screen was arranged by chronological order. it wasn't legible and inefficient because of four different factors unlike alarm clock : Light On / Light Off, Time, Repetition, Top Button / Bottom Button.
