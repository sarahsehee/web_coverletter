Designing Interactive In-App Survey

Overview

Designing interactive in-app lifestyle survey for automatic timer curation / Improved the timer list screen to better legibility

Duration

Feb 2017 - Mar 2017

Challenge

How to install multiple timers that suit users' lifestyle pattern
during the first setup process.

Survey and interview results showed that 94.9% of users set a timer as soon as they paired smartphones and Swithcer devices for the first time. Also, Swithcer's mobile Google Analytics showed that most of the users don't open the mobile app after the first set-up and just stick with a widget on iOS notification center or a widget on Android Wallpaper.

Intention to become a premium user by the number of saved timer

We went back to a group of users who are on the free trial. First, we asked how many timers they save and use and then asked their intention to subscribe one of premium to plan. Surprisingly, the free-to-paid conversion rate become 1.5 times higher if a user set more than three timers.

Based on these two research data, we were exploring ways to help a user set more than three timers on their first Switcher set up process.

Switcher User Persona

Before we get into the sprint, we refined persona to understand the lifestyle and find the right timer use-case patterns for our users.

User's Pain Points

1. User cannot imagine timer use cases that suit in their lifestyle.
2. There are too many steps to save more than one timer.

Design Solution . Version 1

Interactive Lifestyle Pattern Survey

Switcher User cannot find sufficient information about timer in the app. After trying a timer card guide, we realized that pop up modal wasn't enough and it is quite intrusive. So the interactive survey came out as a solution. It must be feeling like a professional consultant, which helps users to make their home smart.

Constraints

The ideal scenario would be incorporate timer function within a widget, but just have a button to open a timer wasn't enough action to encourage users to use the timer. Also, have a timer in widget required more effort and resources than we were limited.

Prototyping and Test

At this early stage of design process, I used ProtoPie, a codeless prototyping tool, to quickly iterate and validate interactions. We've never done any interactive design with long text in mobile app, so it was necessary for us see actionable feedback from user testers.
With the help of protopie, I was able to create seven working prototypes following by three cycles of design-build-test in three days.

Visual Design

With the help of ProtoPie, I didn't need to spend too much time waiting for working prototypes from engineers. There was not any problem with UX writing and and affordance with the wireframe so I moved on to put more visual elements on it. It was supposed to look trustful and lively, but still coherent with Switcher design system.

Timer List Redesign

By adding a full screen interactive survey to shift the focus to users' lifestyle, there must be a change to show the timer.
The timer list screen was arranged by chronological order. it wasn't legible and inefficient because of four different factors unlike alarm clock : Light On / Light Off, Time, Repetition, Top Button / Bottom Button.

Result

So...How did it go?

Unfortunately, we did not implement proper Google Analytics events because of lack of data engineering resources. It is painful for me not to know about the result of the design that we've been carefully crafted.

Although the most tedious, but essential part of design process improved with the use of protoPie. I've never experienced such a fast iteration and smooth transition from design to development with any other tools.

Also, this project made us we think about what the real Smart Home and Device are. Can it be called Smart Home if it requires too much effort and time from users regularly? It gave us a chance to look back user's pain points and focus on problem-solving.
